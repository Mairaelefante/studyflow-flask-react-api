## Suggested GitHub upload strategy

Upload each folder as a separate repository. Make 3–5 commits per repository instead of uploading everything in one commit. Suggested commits:

1. Initial project structure
2. Add data and core logic
3. Add analysis / UI / model / pipeline
4. Improve README and documentation
5. Add validation, tests, or final improvements
